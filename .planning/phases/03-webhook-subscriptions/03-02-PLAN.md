  <files>crates/server/src/webhook/listener.rs, crates/server/src/webhook/worker.rs, crates/server/src/webhook/mod.rs</files>
  <behavior>
    - Listener: NewMessage event with matching subscription (no filter) creates outbox entry
    - Listener: NewMessage event with filter_sender set only matches when sender equals filter
    - Listener: NewMessage event with filter_thread_id set only matches when thread_id equals filter
    - Listener: NewMessage event with both filters set requires both to match
    - Listener: NewMessage event with no matching subscriptions creates no outbox entries
    - Worker: deliver_one sends POST with correct Content-Type, X-Mailrs-Signature, X-Mailrs-Event, X-Mailrs-Delivery headers
    - Worker: successful delivery (2xx) marks outbox entry as delivered
    - Worker: failed delivery marks outbox entry for retry with exponential backoff
    - Worker: entry exceeding max_attempts is marked as permanently failed
  </behavior>
  <action>
    1. Update crates/server/src/webhook/mod.rs to add `pub mod listener;` and `pub mod worker;`

    2. Create crates/server/src/webhook/listener.rs:
       - `pub async fn run(event_bus: &EventBus, pool: &PgPool, mut shutdown: watch::Receiver<bool>)`:
         - Subscribe to event_bus
         - Loop with tokio::select! over rx.recv(), shutdown.changed()
         - On SmtpEvent::NewMessage { user, thread_id, sender, subject, snippet }:
           - Call store::find_matching_subscriptions(pool, &user, "new_message", &sender, &thread_id)
           - For each matching subscription: build WebhookPayload with event="new_message", timestamp=Utc::now().to_rfc3339(), data={user, thread_id, sender, subject, snippet}
           - Call store::enqueue_delivery with serde_json::to_value(&payload) and now as epoch secs
         - On Lagged(n): tracing::warn!("webhook listener lagged, missed {n} events")
         - On other events: skip

    3. Create crates/server/src/webhook/worker.rs:
       - `pub struct WebhookWorker` with fields: pool (PgPool), client (reqwest::Client), poll_interval (u64, default 5 secs), concurrency_limit (usize, default 10)
       - `pub fn new(pool: PgPool) -> Self` — build reqwest::Client with timeout(10s) and connect_timeout(5s)
       - `pub async fn run(&self, mut shutdown: watch::Receiver<bool>)`:
         - Loop with tokio::select!:
           - sleep(poll_interval) => poll_and_deliver
           - shutdown.changed() => return
         - In poll_and_deliver: call store::dequeue_pending(pool, now, 50)
         - For each entry: spawn a delivery task (use tokio::JoinSet or semaphore to limit concurrency)
       - `async fn deliver_one(&self, entry: OutboxEntry) -> Result<(), String>`:
         - Mark inflight
         - Load subscription from DB (query by entry.subscription_id — add store::get_subscription if needed)
         - Serialize payload to bytes
         - Sign with signer::sign_payload(sub.signing_secret.as_bytes(), &payload_bytes)
         - POST to sub.url with headers:
           - Content-Type: application/json
           - X-Mailrs-Signature: sha256={signature}
           - X-Mailrs-Event: new_message (from payload)
           - X-Mailrs-Delivery: {entry.id}
         - If 2xx: mark_delivered
         - Else: mark_failed with error = "HTTP {status}"
         - On reqwest error: mark_failed with error message

    4. Add store::get_subscription(pool, id) -> Result<Option<Subscription>> to store.rs if not already present (needed by worker to get signing_secret)

    5. Add `#[cfg(test)] mod tests` in listener.rs:
       - Test match logic as pure functions. Extract a `pub(crate) fn matches_subscription(sub: &Subscription, sender: &str, thread_id: &str) -> bool` helper so it can be unit tested without DB.
       - Test: no filter matches any sender/thread
       - Test: filter_sender matches only exact sender
       - Test: filter_thread_id matches only exact thread
       - Test: both filters require both to match

    6. Add `#[cfg(test)] mod tests` in worker.rs:
       - Test: deliver_one constructs correct headers (use a mock or test the header building logic as a pure function)
       - Test the retry delay escalation behavior (can reuse store::retry_delay_secs tests)
