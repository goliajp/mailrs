  <files>scripts/init-schema.sql, crates/server/Cargo.toml, crates/server/src/webhook/mod.rs, crates/server/src/webhook/store.rs</files>
  <behavior>
    - create_subscription inserts row and returns id
    - list_subscriptions returns only active subscriptions for a given account
    - delete_subscription soft-deletes by setting active=false, returns true if found
    - find_matching_subscriptions returns subscriptions matching account + optional sender/thread_id filters
    - enqueue_delivery inserts outbox entry with status=pending and returns id
    - dequeue_pending returns entries where status=pending and next_retry <= now, ordered by created_at
    - mark_inflight sets status=inflight
    - mark_delivered sets status=delivered
    - mark_failed increments attempts, sets next_retry with exponential backoff, sets status=failed when max_attempts reached
  </behavior>
  <action>
    1. Add `hmac = "0.12"` to crates/server/Cargo.toml dependencies.

    2. Append to scripts/init-schema.sql the two new tables exactly as specified in 03-RESEARCH.md:
       - `webhook_subscriptions` with columns: id (BIGSERIAL PK), account_address TEXT NOT NULL, url TEXT NOT NULL, event_type TEXT NOT NULL DEFAULT 'new_message', filter_sender TEXT, filter_thread_id TEXT, signing_secret TEXT NOT NULL, active BOOLEAN NOT NULL DEFAULT true, created_at TIMESTAMPTZ NOT NULL DEFAULT now()
       - Indexes: idx_webhook_subs_account on (account_address) WHERE active, idx_webhook_subs_event on (event_type, active) WHERE active
       - `webhook_outbox` with columns: id (BIGSERIAL PK), subscription_id BIGINT NOT NULL REFERENCES webhook_subscriptions(id) ON DELETE CASCADE, payload JSONB NOT NULL, status TEXT NOT NULL DEFAULT 'pending', attempts INTEGER NOT NULL DEFAULT 0, max_attempts INTEGER NOT NULL DEFAULT 8, next_retry BIGINT NOT NULL, last_error TEXT, created_at BIGINT NOT NULL, updated_at BIGINT NOT NULL
       - Index: idx_webhook_outbox_pending on (status, next_retry) WHERE status = 'pending'

    3. Create crates/server/src/webhook/mod.rs:
       - Declare submodules: `pub mod store;` and `pub mod signer;`
       - Define shared types:
         - `Subscription` struct: id, account_address, url, event_type, filter_sender (Option), filter_thread_id (Option), signing_secret, active, created_at
         - `OutboxEntry` struct: id, subscription_id, payload (serde_json::Value), status, attempts, max_attempts, next_retry, last_error (Option), created_at, updated_at
         - `WebhookPayload` struct: event (String), timestamp (String), data (WebhookData)
         - `WebhookData` struct: user, thread_id, sender, subject, snippet
       - All structs derive Serialize, Deserialize as appropriate; query structs use sqlx::FromRow

    4. Create crates/server/src/webhook/store.rs with these pub async functions (all take &PgPool as first arg):
       - `generate_signing_secret() -> String` — 32 random bytes via OsRng, hex-encoded (64 chars)
       - `create_subscription(pool, account, url, event_type, filter_sender, filter_thread_id, signing_secret) -> Result<i64>`
       - `list_subscriptions(pool, account) -> Result<Vec<Subscription>>` — WHERE active = true AND account_address = $1
       - `delete_subscription(pool, id, account) -> Result<bool>` — UPDATE SET active = false WHERE id = $1 AND account_address = $2 AND active = true; return rows_affected > 0
       - `find_matching_subscriptions(pool, account, event_type, sender, thread_id) -> Result<Vec<Subscription>>` — match active subs where (filter_sender IS NULL OR filter_sender = $sender) AND (filter_thread_id IS NULL OR filter_thread_id = $thread_id)
       - `enqueue_delivery(pool, subscription_id, payload, now) -> Result<i64>`
       - `dequeue_pending(pool, now, limit) -> Result<Vec<OutboxEntry>>` — WHERE status = 'pending' AND next_retry <= $now ORDER BY created_at LIMIT $limit
       - `mark_inflight(pool, id, now) -> Result<()>` — UPDATE SET status = 'inflight', updated_at = $now
       - `mark_delivered(pool, id, now) -> Result<()>` — UPDATE SET status = 'delivered', updated_at = $now
       - `mark_failed(pool, id, error, attempt, max_attempts, now) -> Result<()>` — if attempt >= max_attempts: status = 'failed', else status = 'pending' with next_retry = now + retry_delay_secs(attempt). Use same exponential backoff as outbound_queue: base 60s, doubling, capped at 6 hours.
       - `retry_delay_secs(attempt: u32) -> i64` — match outbound_queue pattern: [60, 120, 300, 600, 1800, 3600, 7200, 21600] seconds, capped at 21600

    5. Add `#[cfg(test)] mod tests` in store.rs with unit tests for:
       - `retry_delay_secs` returns correct values for attempts 0-8
       - `generate_signing_secret` produces 64-char hex string
       - Type serialization/deserialization round-trips for WebhookPayload

    1. `type HmacSha256 = Hmac<Sha256>;` (from hmac and sha2 crates)

    2. `pub fn sign_payload(secret: &[u8], payload: &[u8]) -> String`:
       - Create HMAC instance with `HmacSha256::new_from_slice(secret)`
       - Update with payload bytes
       - Finalize and return hex-encoded result (lowercase)

    3. `pub fn verify_signature(secret: &[u8], payload: &[u8], signature: &str) -> bool`:
       - Compute expected signature via sign_payload
       - Use constant-time comparison (hmac's `verify_slice` or compare hex strings via the mac's verify method)
       - Actually, use the hmac crate's built-in verify: create mac, update with payload, then `mac.verify_slice(&hex::decode(signature))` which is timing-safe

    4. `pub fn format_signature_header(signature: &str) -> String`:
       - Returns `format!("sha256={signature}")`

       - Test verify_signature succeeds with correct signature
