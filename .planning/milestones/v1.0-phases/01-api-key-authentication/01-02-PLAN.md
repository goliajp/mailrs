1. In `auth.rs`, add `verify_api_key` function and wire it into `from_request_parts`:

In `from_request_parts`, after extracting the token, add before the session lookup:
```rust
if let Some(ref token) = token {
    if token.starts_with("mlrs_") {
        return verify_api_key(token, state).await;
    }
    // existing session path...
}
```

`verify_api_key` implementation:
- Parse token: split on `_` into 3 parts (mlrs, prefix, secret). If not 3 parts, return 401 "invalid api key format".
- Try Valkey cache: `cache_get(valkey, prefix)`. If no valkey connection, skip to PG.
- Cache miss: query PG via `get_api_key_by_prefix(pool, prefix)`. If no PG pool, return 500 "auth backend unavailable". If no record, return 401.
- On cache miss + PG hit: build `CachedApiKey` (needs account's super_domains — query `domain_store.get_account(&record.account_address)` to get super_domains). Call `cache_set`.
- Verify hash: `sha256_hex(token.as_bytes()) == record.key_hash`. If mismatch, return 401.
- Check expiration: if `expires_at` is Some and < Utc::now(), return 401 "api key expired".
- Fire-and-forget `update_last_used` via `tokio::spawn`.
- Return `AuthUser { address: record.account_address, display_name: account.display_name (or address as fallback), super_domains, auth_method: AuthMethod::ApiKey(record.id) }`.

2. Create `crates/server/src/web/api_key.rs`:

Request/response types:
```rust
#[derive(Deserialize)]
pub struct CreateApiKeyRequest {
    pub name: String,
    #[serde(default)]
    pub expires_at: Option<DateTime<Utc>>,
}

#[derive(Serialize)]
pub struct ApiKeyResponse {
    pub id: i64,
    pub prefix: String,
    pub name: String,
    pub created_at: DateTime<Utc>,
    pub expires_at: Option<DateTime<Utc>>,
    pub last_used_at: Option<DateTime<Utc>>,
}
```

Handlers:

**`create_api_key`** (POST /api/agent/keys):
- Requires session auth only: check `auth_user.auth_method` is `AuthMethod::Session`. If `ApiKey(_)`, return 403 "api keys cannot create other api keys".
- Validate name length (1-100 chars).
- Call `generate_api_key()` to get (full_key, prefix, key_hash).
- Call `insert_api_key(pool, &prefix, &key_hash, &auth_user.address, &req.name, req.expires_at)`.
- If insert fails due to prefix collision (unique constraint), retry up to 3 times with new key.
- Return 201 with JSON: `{ id, key: full_key, prefix, name, expires_at, warning: "Save this key now. It cannot be retrieved again." }`.

**`list_api_keys`** (GET /api/agent/keys):
- Call `list_api_keys(pool, &auth_user.address)`.
- Map to `Vec<ApiKeyResponse>` (never expose key_hash).
- Return 200.

**`revoke_api_key`** (DELETE /api/agent/keys/{id}):
- Extract `id` from path.
- Call `revoke_api_key(pool, id, &auth_user.address)` — account_address constraint ensures users can only revoke their own keys.
- If revoked successfully, also call `cache_delete(valkey, prefix)`. To get the prefix, query the key by id first, OR change revoke_api_key to return the prefix.
  - Better approach: change `revoke_api_key` in api_key_store to return `Option<String>` (the prefix) instead of `bool`. Update the store function to SELECT prefix before UPDATE, or use RETURNING prefix.
- If not found/already revoked, return 404.
- Return 200 with `{ success: true }`.
