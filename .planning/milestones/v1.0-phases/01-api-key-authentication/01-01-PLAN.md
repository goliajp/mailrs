1. Create `scripts/migrate-010-api-keys.sql`:
```sql
CREATE TABLE IF NOT EXISTS api_keys (
    id              BIGSERIAL PRIMARY KEY,
    prefix          TEXT NOT NULL UNIQUE,
    key_hash        TEXT NOT NULL,
    account_address TEXT NOT NULL REFERENCES accounts(address) ON DELETE CASCADE,
    name            TEXT NOT NULL DEFAULT '',
    expires_at      TIMESTAMPTZ,
    last_used_at    TIMESTAMPTZ,
    revoked_at      TIMESTAMPTZ,
    created_at      TIMESTAMPTZ NOT NULL DEFAULT now()
);

CREATE INDEX IF NOT EXISTS idx_api_keys_prefix_active ON api_keys(prefix) WHERE revoked_at IS NULL;
CREATE INDEX IF NOT EXISTS idx_api_keys_account ON api_keys(account_address) WHERE revoked_at IS NULL;
```

2. Create `crates/server/src/api_key_store.rs` with:

- `generate_api_key() -> (String, String, String)` — returns (full_key, prefix, key_hash). Uses `rand_core::OsRng.fill_bytes` for 4 prefix bytes + 20 secret bytes, formats as `mlrs_{8hex}_{40hex}`, SHA-256 hashes the full key.

- `sha256_hex(data: &[u8]) -> String` — uses `sha2::Sha256::digest` + `hex::encode`.

- `ApiKeyRecord` struct with fields matching the DB table (id, prefix, key_hash, account_address, name, expires_at, last_used_at, revoked_at, created_at). Derive `sqlx::FromRow`.

- `CachedApiKey` struct (key_hash, account_address, super_domains: Vec<String>, expires_at, id) with Serialize/Deserialize for Valkey JSON storage.

- `async fn insert_api_key(pool: &PgPool, prefix: &str, key_hash: &str, account_address: &str, name: &str, expires_at: Option<DateTime<Utc>>) -> Result<i64>` — INSERT RETURNING id. On prefix collision (unique constraint violation), return error so caller can retry with new key.

- `async fn get_api_key_by_prefix(pool: &PgPool, prefix: &str) -> Result<Option<ApiKeyRecord>>` — SELECT WHERE prefix = $1 AND revoked_at IS NULL.

- `async fn list_api_keys(pool: &PgPool, account_address: &str) -> Result<Vec<ApiKeyRecord>>` — SELECT WHERE account_address = $1 AND revoked_at IS NULL ORDER BY created_at DESC.

- `async fn revoke_api_key(pool: &PgPool, id: i64, account_address: &str) -> Result<bool>` — UPDATE SET revoked_at = now() WHERE id = $1 AND account_address = $2 AND revoked_at IS NULL, return rows_affected > 0.

- `async fn update_last_used(pool: &PgPool, id: i64)` — UPDATE SET last_used_at = now() WHERE id = $1. Fire-and-forget friendly (no error propagation needed).

- Valkey cache helpers:
  - `async fn cache_get(valkey: &ConnectionManager, prefix: &str) -> Option<CachedApiKey>` — GET "apikey:{prefix}", deserialize JSON.
  - `async fn cache_set(valkey: &ConnectionManager, prefix: &str, cached: &CachedApiKey)` — SET "apikey:{prefix}" with 300s TTL.
  - `async fn cache_delete(valkey: &ConnectionManager, prefix: &str)` — DEL "apikey:{prefix}".

3. Add `pub mod api_key_store;` to `crates/server/src/lib.rs` (or `main.rs` — check which is the crate root).

4. Unit tests in `api_key_store.rs`:

   - `test_sha256_hex`: verify known input produces expected hash
   - `test_generate_key_uniqueness`: generate 100 keys, all prefixes unique

Ensure `cargo build -p mailrs-server` compiles.
